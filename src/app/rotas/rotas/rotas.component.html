Rotas

Routas são caminhos que controlam os componentes renderizados. Para criar um projeto Angular com modulo de roteamento com Angular CLI, execute o comando ng new [app-name] --routing. O modulo de roteamento deve ser importado no modulo raiz do projeto.

Routes e Route

Route é um objeto de configuração que define uma única rota. Um conjunto de rotas é coletado em uma matriz que definir uma configuração de rotas. Essa matriz é definida como Routes.

O objeto Route possui as propriedades path, que representa o caminho da rota, e component, que define o componente que deve ser renderizado. Para aplicar o sistema de roteamento no template (HTML) é necessário definir a tag router-outlet.

Router

"Um serviço que fornece navegação entre visualizações e recursos de manipulação de URL". Este serviço oferece o método navigate() que possibilita a navegação com base em uma matriz de comandos fornecida e um ponto de partida.

Guard

Guards são serviços para controle de acesso as rotas. Para criar um novo Guard com Angular CLI execute o comando ng generate guard [nome do guard] / ng g g [nome do guard].

  • CanActivate (controle de acesso a entrada de rotas)
  • CanActivateChild (controle de acesso a entrada de rotas filhas)
  • CanDeactivate (controle de acesso a saída de rotas)
